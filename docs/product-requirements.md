# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**Zettelkastener** - Zettelkasten専用エディター

### プロダクトコンセプト
- **摩擦ゼロで書き始める**: アイデアが浮かんだ瞬間にショートカット一発で書き始められる。タイトル・分類・保存場所の決断は不要
- **ツールがルールを守る**: 1ノート1アイデア・リンク必須・出典タグ・種別管理を、意志力に頼らずツールが強制・サポートする
- **書くことと整理することを分離する**: Fleeting → Literature → Permanent の流れを自然に促し、キャプチャと整理を同時にやらなくていい設計

### プロダクトビジョン
Zettelkasten方法論に特化したエディターとして、「即座に書ける」と「Zettelkastenのルールを守る」を両立する。
既存ツール（Obsidian, Logseq, Notion）はZettelkastenのルールを何も強制しないため、実践しようとしても規律が崩れる。
Zettelkastenerはノート種別・リンク・出典タグ・1ノート1アイデアをシステムレベルで強制し、実践者が「正しくできているか不安」にならずにZettelkastenを続けられる環境を作る。

### 目的
- ノート作成の摩擦をゼロに近づけ、アイデアを逃さない
- Zettelkastenの原則をツールが担保することで、ユーザーが別途メタ認知しなくてよい状態にする
- 書くことと整理することを分離し、両方を気持ちよくこなせる体験を提供する
- 知識を「保管」するためではなく「生成」するためのネットワークを育てる

---

## ターゲットユーザー

### プライマリーペルソナ: 開発者・Zettelkasten実践者（自分自身）

- Zettelkastenを正しく実践したいが、既存ツールでは規律が崩れた経験がある
- PC（メイン作業）とスマホ（外出時の素早いメモ）の両方を使う
- **現在の課題**:
  - アイデアが浮かんだ瞬間に書き始めるまでの摩擦が大きい（ファイル名・保存場所・テンプレート選択などの決断が多い）
  - 「このアイデアを別のノートと分けるべき？」で迷うたびに思考が止まる
  - 書くことと構造化することを同時にやろうとしてパンクする
  - ツールが何も強制しないので、いつの間にかZettelkastenのルールが崩れる
- **期待する解決策**:
  - 思いついたら即座に書ける（1秒以内にメモ入力モードに入れる）
  - Zettelkastenのルールをツールが強制・サポートしてくれる
  - 書くことに集中できる（整理は後で行う）
  - PCでフル機能を使い、スマホでは手軽にFleeting Noteだけ残せる

## 成功指標 (KPI)

### プライマリーKPI

- **継続率**: 3ヶ月間（90日）のうち75日以上アプリを起動して使用し、Zettelkastenのワークフローを習慣として維持できる（起動率83%以上）
- **摩擦の消滅**: メモ入力モード起動から書き始めるまで1秒以内（計測可能な技術指標）
- **Fleeting Note処理率**: 作成したFleeting Notesの90%以上が7日以内に変換または削除されている

### セカンダリーKPI

- **リンク密度**: 新しいPermanent Noteを作るたびに既存ノートとのリンクが少なくとも1つ自然に見つかる
- **知識の生成**: 月1回以上、ノートを読み返して新しいアイデアや繋がりが生まれる（月次レビューで5段階自己評価4以上を1回以上記録し、補助指標として月間の新規リンク追加数も確認する）
- **ルール遵守率**: 保存されたPermanent Notesの100%が最低1つのリンクを持つ

> **注意**: 「Permanent Notesを週に何枚作れたか」は追わない。枚数より、ノートが知識ネットワークとして機能しているかが本質

### KPI測定方法

| KPI | 測定方法 | 実装方針 |
|-----|---------|----------|
| 継続率 | アプリ起動日数 ÷ 90日間 ≥ 83%（75日以上） | Electronの起動時刻をローカルログに記録 |
| 摩擦の消滅 | ショートカット実行 → 初回文字入力までの時間 | `performance.now()` で計測・ローカルログ記録 |
| Fleeting Note処理率 | 処理済み（変換・削除）ノート数 ÷ 作成後7日経過ノート総数 | `Note.processedAt` フィールドで判定 |
| リンク密度 | 新規Permanent Note作成時のリンク数の平均 | `Link`テーブルのクエリで集計 |
| 知識の生成 | 月次レビューで「新しい繋がりが生まれたか」を5段階で自己評価し、4以上の月が1回以上あるかを確認。補助指標として月間の新規リンク追加数も記録する | 月末にレビュー画面または設定画面から自己評価を記録し、`Link`テーブルから月次追加件数を集計 |
| ルール遵守率 | リンク0件のPermanent Note保存試行が拒否された回数 | バリデーションエラーのログ記録 |

> **補足**: 「知識の生成」は主観的な体験を含むため、自己評価を主指標としつつ、月間の新規リンク追加数を補助指標として併記する。

## 機能要件

### コア機能（P0 / MVP）

---

#### 機能1: 即時メモ入力

**ユーザーストーリー**:
Zettelkasten実践者として、アイデアが浮かんだ瞬間にキャプチャするために、決断なしに即座に書き始められる入力モードが欲しい

**受け入れ条件**:
- [ ] グローバルショートカット一発（例: `Cmd/Ctrl+Shift+N`）でメモ入力モードが起動する
- [ ] 入力モード起動から書き始めるまで1秒以内（タイトル・分類の入力不要）
- [ ] デフォルト種別は「Fleeting Note」に自動設定される
- [ ] そのまま保存するとFleeting Noteとして即保存される
- [ ] Literature / Permanent への昇格は後から行える

**優先度**: P0（必須）

---

#### 機能2: ノート種別の強制と制約

**ユーザーストーリー**:
Zettelkasten実践者として、ルールを自己管理しなくてよいように、種別ごとの保存制約をツールが強制してほしい

**受け入れ条件**:
- [ ] ノートは必ず Fleeting / Literature / Permanent / Index (MoC) のいずれかの種別を持つ
- [ ] Fleeting Note は制約なしに即保存できる
- [ ] Literature Note は出典タグなしでは保存できない
- [ ] Permanent Note はリンクが最低1つなしでは保存できない
- [ ] 種別ごとにUIが明確に区別されている（色・アイコン等）

| 種別 | 保存制約 | 説明 |
|---|---|---|
| Fleeting Note | なし | とりあえずのメモ。処理後は原則削除 |
| Literature Note | 出典タグ必須 | 文献を読みながら自分の言葉で書いたメモ |
| Permanent Note | リンク最低1つ必須 + 自律性確認プロンプト | 単独で理解できる恒久ノート |
| Index (MoC) | Permanentと同様の制約 | Permanent Notesへのリンク集。テーマ別の入り口ノート |

> **注意**: Literature Notesは書誌情報の管理ツールではない。文献を読みながらその場で触発されて考えたことを書く場所。出典タグはその思考の根拠として付与する
>
> **実装スコープの補足**: `Index (MoC)` 種別そのものと保存制約はP0で定義する。一方で、索引ノートを作成・一覧・探索しやすくする専用UI導線はP1の機能11で実装する。

**優先度**: P0（必須）

---

#### 機能3: 1ノート1アイデアの補助

**ユーザーストーリー**:
Zettelkasten実践者として、1ノート1アイデアの原則を守るために、ノート保存時にアイデアの単一性を確認する仕組みが欲しい

**受け入れ条件**:
- [ ] Permanent Note（または昇格時）の保存時に「このノートで主張していることは何個ですか？」と問いかける
- [ ] 複数と答えた場合は分割フローを提案する
- [ ] 分割フロー: 選択した範囲を新しいノートとして切り出せる
- [ ] Fleeting Note 保存時は問いかけをしない（即保存を妨げない）

> **注意**: 文字数ベースの警告はしない。長くても1アイデアなら問題なく、短くても複数アイデアを含めば違反

**優先度**: P0（必須）

---

#### 機能4: リンク管理の強制とサジェスト

**ユーザーストーリー**:
Zettelkasten実践者として、Permanent Noteが必ず知識ネットワークに接続されるために、リンクの入力を強制・補助してほしい

**受け入れ条件**:
- [ ] Permanent Note はリンクが0件では保存できない（保存時にバリデーション）
- [ ] ノート編集中にタイトル・タグから自動でリンク候補をサジェストする
- [ ] `[[` または指定のショートカットでリンク候補ポップアップが開く
- [ ] リンク先ノートへワンクリック/キーボードでジャンプできる
- [ ] 双方向リンク：リンク先ノートからも参照元が確認できる

**優先度**: P0（必須）

---

#### 機能5: Permanent Noteの自律性確認

**ユーザーストーリー**:
Zettelkasten実践者として、Permanent Noteが文脈なしに単独で理解できることを確認するために、保存時にチェックしてほしい

**受け入れ条件**:
- [ ] Permanent Note 保存時に「このノートは他のノートを参照せずに単独で理解できますか？」の確認プロンプトを表示する
- [ ] Yes / No を選択できる
- [ ] No を選択した場合「文脈の補足をノート本文に追記するか、分割することを検討してください」のガイドを表示する
- [ ] 強制ブロックはしない（ユーザーの意識付けが目的）

**優先度**: P0（必須）

---

#### 機能6: Luhmann式ID採番（Folgezettel）

**ユーザーストーリー**:
Zettelkasten実践者として、思考の連鎖を記録するために、既存ノートへの続きとしてIDを付番できる仕組みが欲しい

**受け入れ条件**:
- [ ] **Permanent Noteにのみ適用**。Fleeting / Literature はIDを持たない
- [ ] Permanent Note 作成時または昇格時に「このノートはどのノートから派生しましたか？」を選択できる
- [ ] 派生元を選ぶと自動でIDが決まる（例: 派生元が `1` なら `1a`、`1a` なら `1a1`、`1` の別展開なら `1b`）
- [ ] 派生元なしで作成した場合はトップレベルの連番（例: `2`, `3`, ...）
- [ ] IDはノートのタイトル横に表示される

**優先度**: P0（必須）

---

#### 機能7: 出典タグ（Literature Notes）

**ユーザーストーリー**:
Zettelkasten実践者として、Literature Noteの出典を管理するために、タグ形式で出典を付与できる仕組みが欲しい

**受け入れ条件**:
- [ ] 著者名・タイトル・URL等をタグ形式で付与できる（例: `@author:Ahrens`, `@title:How to Take Smart Notes`）
- [ ] 出典タグなしでは Literature Note として保存できない
- [ ] 同じ出典タグを持つノートを一覧できるフィルタリング機能を提供する
- [ ] 出典タグはノートのメタデータ領域に表示され、メインコンテンツ（自分の言葉のメモ）と区別される

**優先度**: P0（必須）

---

### 重要機能（P1）

---

#### 機能8: スマホ対応（Fleeting Notes専用）

**ユーザーストーリー**:
外出中のZettelkasten実践者として、外出先でもアイデアをキャプチャするために、スマホからFleeting Noteだけ素早く入力できる軽量UIが欲しい

**受け入れ条件**:
- [ ] スマホブラウザから動作するUI（PWA）。対象ブラウザ: iOS Safari 17+ / Android Chrome 124+（またはリリース時点の最新安定版）
- [ ] Fleeting Note の入力・保存のみに特化したシンプルUI
- [ ] 起動から入力開始まで2秒以内
- [ ] PCとのデータ同期は手動転送でも可（MVP段階では自動同期は不要）

> **補足**: P1として実装候補に含めるのは「スマホ向け軽量PWA UI」。スコープ外なのはスマホとPCの自動同期機能のみ。

**優先度**: P1（重要）

---

#### 機能9: リンクグラフの可視化

**ユーザーストーリー**:
Zettelkasten実践者として、知識ネットワークの全体像を把握するために、ノート間の繋がりをグラフで可視化してほしい

**受け入れ条件**:
- [ ] Permanent Note間のリンクをノードとエッジで表示する
- [ ] グラフ上のノードをクリックすると該当ノートを開ける
- [ ] リンクのないノート（孤立ノート）を視覚的に識別できる（色・アイコン等）
- [ ] テーマ・タグでフィルタリングできる

**優先度**: P1（重要）

---

#### 機能10: Fleeting Notesのレビューフロー

**ユーザーストーリー**:
Zettelkasten実践者として、Fleeting Notesが溜まらないように、定期的に処理するレビューフローが欲しい

**受け入れ条件**:
- [ ] 未処理のFleeting Notes一覧を表示するレビュー画面がある
- [ ] **「Permanent/Literature Noteに変換して削除する」がデフォルトの処理**（削除せずに残すのは例外）
- [ ] 変換フロー: Fleeting Noteの内容をもとに新しいPermanent/Literature Noteを作成し、元のFleeting Noteを削除する
- [ ] 7日以上処理されていないFleeting Notesにバッジ・警告を表示する（ホーム画面等で目に入る場所）

**優先度**: P1（重要）

---

#### 機能11: 索引ノート・構造ノート（Map of Content）

**ユーザーストーリー**:
Zettelkasten実践者として、ノート数が増えても特定のトピックを見つけられるように、テーマ別の入り口ノートを作りたい

**受け入れ条件**:
- [ ] 「索引ノート（Map of Content）」という特別な種別のノートを作成できる
- [ ] 索引ノートは主にPermanent Notesへのリンク集として機能し、必要に応じてトピックの説明も書ける
- [ ] 索引ノートはグラフ表示上でハブとして視覚的に識別できる
- [ ] ノート一覧で索引ノートに素早くアクセスできる導線がある

**優先度**: P1（重要）

---

#### 機能12: 全文検索

**ユーザーストーリー**:
Zettelkasten実践者として、過去のノートを素早く参照するために、タイトル・本文・タグ・IDを横断検索できる機能が欲しい

**受け入れ条件**:
- [ ] タイトル・本文・タグを横断検索できる
- [ ] Luhmann式IDで直接検索できる
- [ ] 検索結果をノート種別でフィルタリングできる
- [ ] 100ノート規模で検索結果表示が1秒以内

**優先度**: P1（重要）

## 非機能要件

### パフォーマンス
- メモ入力モードの起動: **1秒以内**（「即座に書ける」の核心。超えると体験が壊れる）
- Permanent Note 保存: 体感で即座（UI操作完了から500ms以内）
- 全文検索: 100ノート規模で結果表示1秒以内
- グラフ表示: 500ノート規模で初期描画3秒以内

#### パフォーマンステスト基準

詳細な計測環境・負荷条件・テスト方法は [docs/architecture.md](architecture.md#パフォーマンス要件) を参照する。本ドキュメントでは要求レベルの目標値のみを管理する。

### ユーザビリティ
- キーボードショートカットで主要操作（新規作成・保存・リンク挿入・検索）が完結する
- 制約違反時のエラーメッセージは「何が問題で、どう解決するか」を具体的に示す

#### ユーザビリティ受け入れ基準

**タスク**: 初見ユーザーがFleeting → Permanent の基本ワークフローを完了する

**手順**:
1. アプリを起動し、ショートカットでFleeting Noteを作成・保存する
2. レビューフローでFleeting Noteを開く
3. Permanent Noteに昇格し、リンクを1つ追加して保存する

**目標時間**: 15分以内（オンボーディング画面またはドキュメントを参照可）

**測定方法**: Zettelkasten未経験のPC/スマホユーザー3名でテスト、平均完了時間を計測（テスター属性: Zettelkasten未経験、PC日常使用者）

### 信頼性
- ノートデータが消えない（保存操作の失敗時もデータを保持する）
- 入力中の内容は30秒ごとに自動保存される下書きとして保持される
- アプリのクラッシュ後に再起動した際、書きかけのノートが復元できる

### スコープ外（MVP段階）

- スマホとPCの自動同期機能（機能8の軽量PWA UI自体は対象候補、除外対象は自動同期のみ）
- AI/LLMによるリンク候補自動提案・1ノート1アイデアの自動判定
- 複数ユーザーによるコラボレーション機能
- クラウドホスティング・バックアップ
- モバイルネイティブアプリ（iOS/Android）
- Windowsプラットフォーム対応（macOS・Linuxのみ対応）

## ワークフロー定義

### 基本ワークフロー: Fleeting → Permanent

```
[外出先 or 思いついた瞬間]
  ↓
1. ショートカット一発でメモ入力モードを起動
2. Fleeting Noteとして即保存（制約なし）
  ↓
[後で時間があるとき: レビューフロー]
  ↓
3. Fleeting Note一覧からレビュー対象を選ぶ
4a. Permanent Noteに変換:
    - 本文を自分の言葉で書き直す（自律性を持たせる）
    - 既存ノートへのリンクを最低1つ追加
    - 自律性確認プロンプトに回答
    - Folgezettelとして派生元を選択（IDが決まる）
    - 元のFleeting Noteを削除
4b. Literature Noteに変換:
    - 文献を読みながら書いたメモとして整理
    - 出典タグを付与
    - 元のFleeting Noteを削除
4c. 不要と判断したら削除
```

### ワークフロー: Literature → Permanent

```
[文献を読みながら]
  ↓
1. Literature Noteとして出典タグ付きで保存（読みながら書く）
  ↓
[後でアイデアを昇華させるとき]
  ↓
2. Literature Noteを元にPermanent Noteを派生作成
3. 自分の言葉で書き直し、自律した考えとして定式化
4. 既存Permanent Notesへのリンクを追加
```
